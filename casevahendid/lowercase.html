Lower-CASE vahendid

Lower-CASE (Computer-Aided Software Engineering) vahendid on tööriistad, mis keskenduvad tarkvaraarenduse hilistele etappidele: koodi kirjutamine, testimine, silumine, versioonihaldus ja juurutamine. Need toetavad otseselt arendajaid tehnilises teostamises.

Mis on Lower-CASE vahendid ja mille jaoks neid kasutatakse?

Lower-CASE tarkvaraarendusprotsessides tähendab süsteemi madala taseme modelleerimist ja tehnilist teostust. Need vahendid aitavad automatiseerida ja hõlbustada tehnilisi protsesse.

  • Git ja GitHub - Versioonihalduseks ja koodi koostööks
  • JUnit - Java testimisraamistik, automatiseeritud testimise toetamiseks
  • Selenium - Veebirakenduste kasutajaliidese testimiseks
  • Jenkins - Pideva integreerimise ja juurutamise automatiseerimiseks
  • Visual Studio Code / IntelliJ IDEA - Integreeritud arenduskeskkonnad (IDE)
  • Postman - API-de testimiseks ja dokumenteerimiseks

Milliseid vahendeid olen ise kasutanud ja milliseid ei ole?

  • Kasutanud: Git ja GitHub - Neid olen kasutanud versioonihalduseks ja projektide jagamiseks
  • Ei ole kasutanud: JUnit - Seda ei ole kasutanud
  • Ei ole kasutanud: Selenium - Seda ei ole kasutanud
  • Ei ole kasutanud: Jenkins - Seda ei ole kasutanud

Lower-CASE vs Upper-CASE

Erinevalt Upper-CASE vahenditest, mis keskenduvad süsteemi analüüsile, disainile ja modelleerimisele (nagu UML-diagrammid), on Lower-CASE vahendid rohkem tehnilised ja koodikesksed. Need täiendavad üksteist terviklikus arendusprotsessis.
